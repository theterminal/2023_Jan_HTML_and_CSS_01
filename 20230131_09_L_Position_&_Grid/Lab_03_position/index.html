Static

'position: static'

'position: static;' is the default position value that every element is created with, regardless of the type of the element. It is NOT necessary to declair it because it is its default value.

In 'position: static;' mode, the element cannot be positioned! The value must be changed to another positon value (relative, absolute, fixed or sticky).

When the element is inspected it is visible that, for example, the properies 'top: 50px' and 'left: 50px' are set but they do not have an affect on positioning the image.

Example for 'position: static;'

.class-exmple: {
position: static;
top: 50px;
left: 50px;

Relative

'position: relative;'

When using 'position: relatve;' the flow of our HTML stays unchanged. And the declarations of 'top', 'bottom', 'left'... just position our element relative to its original (initial static) position. All elements, their margins, paddings etc. stay same as if the position of our element is 'position: static;

Example for 'position: relative;'

.class-exmple: {
position: relative;
top: 50px;
left: 50px;
}

'top: 50px;' - this means that our actual element will be positiond 50px DOWN from the top of where it would be if the declaration was 'position: static'

The values can be positive or negative.

Absolute

'position: absolute;'

To use 'position: absolute;' the rule in the image CSS must be set to 'position: absolute;' AND the container of the element should have the property'position: relative;' set. If we do NOT have that the image will search UP the tree until it finds container with position DIFFERENT than 'static' and will position the image according to that element.

'position: absolute' creates a layer above the original layer and the image element or the element we need to position absolute occupies that new layer. The original layer changes the flow of the HTML elements.

When using negative values and the element leaves the container borders, the part that is out of the container can be hidden with the 'overflow: hidden:' rule.

Fixed

'position: fixed'

Position 'fixed' just as the 'absolute' takes the element out of the document flow.

'position: fixed' is positioning the element ALWAYS to the HTML (browser). It is NOT necessary the container of the element to have 'position: relative' or other.

Example for 'position: fixed;' (it is a round image at the right bottom of the page)

.fixed{
background-color: lightpink;
position: relative;
}

.fixed img{
width: 50px;
height: 50px;
border-radius: 100%;
position: fixed;
bottom: 20px;
right: 20px;
}

Sticky

'position: sticky'

'position: stiky' makes the element to stick at the spesified position and to stay there while the viewer scrolls out of the container on the screen. The element DOES NOT leave the border of the container where it is created...

Example for 'position: sticky;'

.sticky{
background-color: lightsalmon;
/* height: 400px; */
/* overflow: auto; */
}

.sticky p{
margin: 25px 0px;
}

.sticky img{
position: sticky;
top: 0px;
}

The aboive CSS declarations are for the current image, box, container. You can look at the actual CSS and see that it is the same.
